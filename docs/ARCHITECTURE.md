# 架构设计文档

## 概述

Map Navigation App 是一个基于 PyQt5 的桌面地图导航应用，采用分层架构设计。

## 架构图

```
┌─────────────────────────────────────────┐
│              表示层 (UI)                 │
│  ┌─────────────┐    ┌─────────────────┐ │
│  │  map.py     │    │  Mainwce.py     │ │
│  │  主窗口     │    │  辅助UI组件     │ │
│  └─────────────┘    └─────────────────┘ │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              业务逻辑层                  │
│         ┌─────────────────┐             │
│         │   dis33.py      │             │
│         │  路径算法模块   │             │
│         └─────────────────┘             │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│              数据访问层                  │
│      ┌──────────────────────┐           │
│      │  database_option.py  │           │
│      │   SQLite 数据库      │           │
│      └──────────────────────┘           │
└─────────────────────────────────────────┘
```

## 模块职责

### 1. 表示层 (Presentation Layer)

**map.py**
- 主窗口实现
- 用户交互处理
- 界面状态管理
- 信号槽连接

**Mainwce.py**
- 辅助 UI 组件
- 自定义控件
- 对话框实现

### 2. 业务逻辑层 (Business Logic Layer)

**dis33.py**
- 图的表示和构建
- 最短路径算法实现
- 路径结果格式化

### 3. 数据访问层 (Data Access Layer)

**database_option.py**
- 数据库连接管理
- 用户数据 CRUD
- 数据验证

## 数据流

### 用户登录流程

```
用户输入 -> UI验证 -> 数据库查询 -> 结果返回 -> UI显示
```

### 路径规划流程

```
选择起点/终点 -> 构建图 -> 运行算法 -> 返回路径 -> 显示结果
```

## 设计模式

### MVC 模式

- **Model**: database_option.py, dis33.py
- **View**: map.py, Mainwce.py
- **Controller**: 内嵌在 UI 类中

### 单例模式

数据库连接使用单例模式管理。

## 扩展性

### 添加新算法

1. 在 `dis33.py` 中实现新算法
2. 在 UI 中添加算法选择选项
3. 更新文档

### 添加新数据源

1. 扩展 `database_option.py`
2. 添加新的数据库表
3. 更新相关 UI

## 性能考虑

### 数据库
- 使用连接池（如需要扩展）
- 索引优化查询

### 算法
- Floyd-Warshall 适合小规模图
- Dijkstra 适合单源最短路径
- 考虑使用 A* 算法优化大场景

## 安全考虑

### 密码存储
- 当前：明文存储（开发测试用）
- 建议：使用哈希算法（如 bcrypt）

### 输入验证
- 防止 SQL 注入
- 输入长度限制
