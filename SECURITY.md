# Security Policy

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 支持 |
| < 1.0 | ❌ 不支持 |

## 报告安全漏洞

如果您发现了安全漏洞，请通过以下方式报告：

1. **不要** 创建公开的 GitHub Issue
2. 发送邮件至仓库维护者
3. 或联系项目管理员

## 安全更新

安全更新将：
- 尽快发布
- 在 CHANGELOG.md 中记录
- 标记为安全修复

## 安全实践

### 当前实现

- 用户密码以明文存储（开发测试用途）
- SQLite 数据库本地存储

### 建议改进

- 使用 bcrypt 等算法哈希存储密码
- 实施输入验证防止 SQL 注入
- 使用 HTTPS（如部署为 Web 应用）
- 定期更新依赖库

## 已知问题

暂无已知安全问题。

如有安全相关问题，请及时联系维护者。
